## What This Project Is

A general-purpose multi-agent conversation experiment engine. LLM-powered agents embodying historical intellectual personas discuss unsolved problems across structured rounds. The system is config-driven — new experiments require zero code.

The key research question: does cross-domain collaboration (e.g. physicists + philosophers) produce qualitatively different discourse than domain-internal groups? Does a translation layer (the MetaCommunicator) enable genuine intellectual exchange?

## Core Concepts You Must Understand

### MACP Protocol
Agents communicate through a Hub in synchronous rounds. Each round: all agents receive the broadcast → think privately → speak or pass → MetaCommunicator translates between domain groups → Hub assembles split broadcasts → Judge evaluates → CONTINUE or CLOSE.

### MetaCommunicator
When enabled, translates messages between domain groups. NOT a generic simplifier — audience-aware, builds bridges using the receiver's own vocabulary. Called once per sender/receiver pair, bidirectionally.

### Agent Memory
Two layers: (1) **Reflections** — concise semantic notes generated by the agent after each round, embedded locally and retrieved by cosine similarity before the next round. (2) **Full round archive** — verbatim JSONL on disk, never fed back to agents during experiment, preserved for analysis.

### Checkpoint System
Every phase of every round is saved atomically before the next API call. On restart: reads `session_state.json` + round JSONL files → skips completed phases → resumes from first incomplete. No data is ever re-run.

### Config Loading
`ConfigLoader(experiment_dir, project_root)` loads `experiment.json` and resolves all file references. Agent personas can be in the experiment's own `personas.json` or referenced from `shared/personas/`. Domains and names must match exactly.

## How to Make a Specific Change

### Adding a new provider (e.g. OpenRouter)
1. Add to `PROVIDER_BASE_URLS` in `core/src/api/client.py`
2. Add `"openrouter": {"keys": [...]}` to `config/api_keys.json`
3. Add models to `config/rate_limits.json`
4. Add reset config to `RESET_CONFIG` in `core/src/api/rate_tracker.py`

### Adding a new persona
Add to the relevant domain file in `shared/personas/` or to the experiment's own `personas.json`. No code change.

### Adding a new experiment
Copy `experiments/_template/`, edit the JSON files, run `validate.py`. No code change.

### Changing how translations work
Edit `core/src/agents/meta_communicator.py`. The prompt template is `META_COMM_TASK_TEMPLATE`. The group logic is in `run_translations()`.

### Changing how the Judge decides
Edit the `system_prompt` in `experiments/*/judge.json` or `experiments/_template/judge.json`. For code changes to how Judge output is parsed, edit `core/src/protocol/judge.py`.

### Changing how reflections are generated
Edit `core/src/memory/reflector.py`. The prompt suffix is `REFLECTION_SYSTEM_SUFFIX`. JSON parsing is in `_parse_reflection_json()`.

### Changing the round execution flow
**Read `core/src/protocol/round_runner.py` extremely carefully before touching it.** Phases must stay in order and every phase must append its event before moving forward. Breaking this breaks the atomic resume guarantee.

### Adding evaluation/analysis capabilities
The full data is in `experiments/*/sessions/run_N/rounds/*.jsonl`. Each file contains one JSON event per line. Event types: `round0_delivery`, `agent_output`, `meta_comm_translation`, `broadcasts_assembled`, `reflection_generated`, `judge_evaluated`. You can read and analyse these independently of the running system.

## What Has NOT Been Built Yet

- **Analysis tooling** — no scripts yet for reading and analysing JSONL session data. This is the next major piece. The data is all there, just needs tooling.
- **Evaluation framework** — no automated quality metrics for the conversations. Planned as a separate Claude task.
- **More than 2 providers** — only Groq and Google AI Studio are implemented.
- **4th physicist persona** — `experiments/quantum_gravity_control` currently has 3 agents. User intends to add a 4th physicist to `shared/personas/theoretical_physics.json`.

## File Modification Safety Guide

| File | Safe to edit? | Risk if wrong |
|---|---|---|
| `experiments/*/experiment.json` | ✓ freely | Config conflict dialog prompts user on next run |
| `experiments/*/personas.json` | ✓ freely | Validation catches errors before running |
| `config/api_keys.json` | ✓ freely | Keys validated at startup |
| `config/rate_limits.json` | ✓ carefully | Wrong values affect quota tracking |
| `core/src/utils/checkpoint.py` | ✗ avoid | Breaks atomic resume if changed incorrectly |
| `core/src/protocol/round_runner.py` | ✗ avoid | Phase ordering is critical |
| `core/src/utils/config_loader.py` | ✓ carefully | Changes affect all experiments |
| All other core modules | ✓ carefully | Test after changes |

## Session Data Format

Each round file: `experiments/*/sessions/run_N/rounds/round_N.jsonl`

Read with:
```python
import json
events = [json.loads(line) for line in open("round_3.jsonl")]

# Phase breakdown
agent_outputs  = [e for e in events if e["event_type"] == "agent_output"]
translations   = [e for e in events if e["event_type"] == "meta_comm_translation"]
reflections    = [e for e in events if e["event_type"] == "reflection_generated"]
judge          = next(e for e in events if e["event_type"] == "judge_evaluated")

# Agent message
msg = agent_outputs[0]["output"]["message"]   # None if passed
passed = agent_outputs[0]["output"]["passed"]  # bool

# Translation
t = translations[0]
print(t["from"], "→", t["for"])
print("Original:  ", t["original"][:100])
print("Translated:", t["translated"][:100])

# Judge
print(judge["judge"]["decision"])      # CONTINUE or CLOSE
print(judge["judge"]["observation"])   # Why
```

## Conversation Context for This Claude Instance

This document was created as part of a long conversation designing and building the DiverseIntelligence system from scratch. Key decisions made during that conversation:

- **MetaCommunicator** (renamed from "The Interpreter") does audience-aware bidirectional translation between groups defined in config
- **Agent memory** uses reflection (semantic notes generated post-round, embedded locally) + full verbatim archive on disk. No summarisation — reflections replace it.
- **Embedding model**: `all-MiniLM-L6-v2`, local CPU, ~80MB, no API calls
- **Experiment config** is the user-facing surface — hyperparameters, agent lists, models, temperatures are all in JSON. Code is hidden behind the `core/` engine.
- **Multiple API keys per provider** are supported and rotated automatically
- **Rate limits**: Groq resets midnight UTC, Google resets midnight Pacific
- **UI**: Textual TUI (`ui.py`). Experiment screen has Overview / Hyperparameters / Run / Live Logs tabs. Can launch headless (close UI, run in terminal) or with live log streaming.
- **Temperature scheduling**: agents vary per-run (list in config), other roles are fixed scalars
